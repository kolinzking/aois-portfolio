# v34 Introduction

Authoring status: authored

## What This Version Is About

v34 completes Phase 10 by adding governance verification for computer-use
actions. It asks whether AOIS-P can prove that a proposed action is authorized,
bounded, observable, reversible, stoppable, red-team-cleared, release-approved,
and access-approved before anything touches a real environment.

## Why It Matters In AOIS

Computer use can change state. It can click, type, submit, move data, expose
credentials, or trigger external commitments. That makes it different from a
text answer or even a tool plan. Governance must run before action, not after
something has already happened.

## How To Use This Version

Use the v34 plan to evaluate local synthetic computer-use plans only. The
lesson does not start a browser, capture screenshots, click, type, submit
forms, access credentials, call providers, or perform external actions.

Every allowed path needs policy, target, approval, preview, budget, stop,
rollback, audit, redaction, operator watch, red-team clearance, release, and
access controls.
