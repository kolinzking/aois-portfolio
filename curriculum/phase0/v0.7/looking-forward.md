# v0.7 Looking Forward

Authoring status: authored

## What You Should Carry Forward

- provider calls are operational boundaries, not casual script details
- token budget affects cost and latency
- structured output is safer for AOIS automation
- API keys require explicit approval and safe storage
- dry-run design reduces waste before real integration
- rough estimates are useful for planning but not exact billing

## What The Next Version Will Build On

`v0.8` will build persistence foundations.

The next step is to store incidents and analysis results in a database-friendly shape so AOIS can move from one-off terminal output toward durable infrastructure behavior.

Do not move forward until you can explain why a planned model call and a real provider call are different operational events.
