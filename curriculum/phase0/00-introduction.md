# Phase 0 Introduction

Phase 0 establishes the operating habits for the rest of AOIS.

The goal is not to collect disconnected scripts. The goal is to build the foundation of one continuous system with:

- disciplined local execution
- clear version boundaries
- explicit reasoning for each tool
- small, reviewable progress

## What Belongs In Phase 0

Examples of valid Phase 0 work:

- shell scripts
- Linux inspection and automation tasks
- Git workflow practice
- basic HTTP and Python work
- early application scaffolding only when needed

## What Does Not Belong Yet

Avoid introducing complex platform layers too early:

- no premature microservice breakdown
- no empty infrastructure directories for future ideas
- no persistent agent memory files unless explicitly requested

## Documentation Standard

Each version should leave behind notes that answer:

1. What was built?
2. Why was it built?
3. What commands were run?
4. What output or behavior mattered?
5. What broke and how was it fixed?
6. What should happen next?

This keeps the repo useful as both curriculum and portfolio proof.
