# v0.3 Lab

## Build Lab

Run:

```bash
git status --short
git log --oneline --decorate -8
```

Write down what the current repository state tells you.

## Ops Lab

Run:

```bash
git status --short
git diff -- README.md || true
git log --oneline --decorate -8
```

Write down:

- what is uncommitted
- what is already protected in history
- what the last few commits say about the AOIS journey

## Break Lab

Do these concept drills:

1. Describe what can go wrong if you commit without reading status.
2. Explain why mixing unrelated files into one commit weakens the portfolio.
3. Compare a vague commit message to a precise one and explain the difference.

## Explanation Lab

Answer the four-layer rule for:

- `git status`
- `git add`
- `git commit`
- `git log`

Then explain `v0.3` at:

1. simple English
2. practical level
3. technical level
4. engineer level

## Defense Lab

Answer:

Why is Git a core AOIS skill and not just a supporting utility?
