# v0.3 Introduction

Authoring status: authored

## What This Version Is About

`v0.3` is Git as engineering memory.

This version is about learning how to make AOIS history inspectable instead of accidental.

You will practice:

- checking repo state
- reading diffs
- staging intentionally
- committing coherent work
- inspecting recent history

## Why It Matters In AOIS

AOIS will become too large to manage by memory.

If Git is weak, later work becomes unsafe:

- scripts change without explanation
- APIs evolve without history
- infrastructure changes become hard to audit
- AI behavior changes become hard to trace

Git is how the repo proves its own evolution.

## How To Use This Version

1. read the notes in order
2. run the Git commands from the repo root
3. inspect before staging
4. commit only coherent work
5. do the break lab on purpose
6. do not move on until Git status and diff feel readable
