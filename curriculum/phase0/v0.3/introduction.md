# v0.3 Introduction

`v0.3` looks less exciting than Linux or log analysis if you think only about runtime features.

That is a mistake.

AOIS is not just a program.
It is also:

- an engineering history
- a recoverable system
- a reviewable portfolio

This version makes Git part of the system-building discipline from the start.

## What You Will Build

You will not build a new runtime component here.
You will build a reliable development workflow:

- inspect repo state
- stage changes deliberately
- commit clean snapshots
- read history as engineering memory

## Why This Version Exists

Later phases will add:

- more files
- more services
- more infra
- more mistakes
- more need for recovery and proof

Without Git discipline, the curriculum becomes harder to trust and AOIS becomes harder to defend in interviews.
