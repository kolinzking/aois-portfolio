# v0.8 Looking Forward

Authoring status: authored

## What You Should Carry Forward

- persistent systems need schema discipline
- server-visible database objects must be unambiguous
- `aois_p` is the portfolio namespace for future database work
- constraints protect data quality
- foreign keys preserve relationships
- provider-call permission should be explicit and auditable
- database runtime requires resource approval on this server

## What The Next Version Will Build On

The Phase 0 capstone will build on the full chain:

`Linux -> Bash -> Git -> HTTP -> Python -> API -> LLM dry-run -> schema`

Do not move forward until you can explain how an incident travels from raw message to analysis result to future durable record.
