# v0.5 Introduction

`v0.5` is where AOIS stops being mostly shell-driven and starts becoming structured Python.

This version is not about "learning Python" in the abstract.
It is about adopting the specific patterns the rest of AOIS will keep using:

- dependency files
- environment loading
- typed models
- small reusable modules

## What You Will Build

You will work through:

- [requirements.txt](../../../requirements.txt)
- [.env.example](../../../.env.example)
- [app/config.py](../../../app/config.py)
- [app/models.py](../../../app/models.py)

## Why It Matters In AOIS

Later versions need:

- validated request bodies
- predictable config loading
- shared models
- code that can move into services and agents cleanly

`v0.5` lays that base.

## How To Use This Version

Start with [notes.md](notes.md), run [lab.md](lab.md), keep [runbook.md](runbook.md) as the quick reference, then close with [summarynotes.md](summarynotes.md) and [looking-forward.md](looking-forward.md).
