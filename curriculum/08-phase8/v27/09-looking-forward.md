# v27 Looking Forward

Authoring status: authored

## What You Should Carry Forward

Carry forward these Phase 8 outcomes:

- v26 product visibility contract
- v27 identity-aware access contract
- explicit dashboard resources
- role and permission matrix
- tenant-scoped resource ownership
- redaction gates
- approval separation
- limited break-glass access
- audit events for allowed and denied decisions

## What The Next Version Will Build On

v28 starts Phase 9: delivery pipeline and release controls.

The next version builds on the Phase 8 product surface by asking how AOIS-P
changes reach users safely. Access policy is not complete if unsafe releases
can bypass tests, signatures, release gates, rollout controls, model rollout
controls, or feature flags.
