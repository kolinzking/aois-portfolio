# v13.5 Looking Forward

Authoring status: authored

## What You Should Carry Forward

Carry forward these rules:

1. GPU workloads need resource advertisement.
2. Scheduling controls are part of reliability.
3. Observability must connect pods to GPU usage.
4. MIG needs a strategy, not guesswork.
5. No operator or plugin is installed without approval.

## What The Next Version Will Build On

`v14` builds on this by moving into high-throughput inference serving.

The next question is:

How does AOIS serve more tokens with controlled latency, concurrency, and cost?
