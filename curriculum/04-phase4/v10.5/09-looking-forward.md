# v10.5 Looking Forward

Authoring status: authored

## What You Should Carry Forward

Carry forward three rules:

1. Orchestration ownership matters.
2. Tool permissions must be explicit before agents act.
3. Provider-managed convenience must be balanced against auditability, rollback, data boundaries, and coupling.

Do not evaluate a managed agent only by how easy it is to create.

## What The Next Version Will Build On

`v11` builds on this by planning event-driven cloud workflows.

The same question remains active:

What does AOIS own, what does a provider own, and how can the decision be validated before anything live is created?
