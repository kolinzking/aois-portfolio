# v12 Looking Forward

Authoring status: authored

## What You Should Carry Forward

Carry forward these rules:

1. Managed runtime is not automatically governed runtime.
2. Workloads should use workload identity, not static keys.
3. Observability must exist before live operation.
4. Cost limits must exist before provider capacity exists.
5. `aois-p` naming protects primary AOIS separation.
6. Cloud execution remains approval-gated.

## What The Next Version Will Build On

Phase 5 builds on this by moving into data and retrieval infrastructure.

The next question is:

How does AOIS manage knowledge, retrieval, and data boundaries with the same rigor it now applies to runtime infrastructure?
