# v19 Looking Forward

Authoring status: authored

## What You Should Carry Forward

Carry forward these rules:

- Do not run chaos when steady state is broken.
- Do not run chaos without a hypothesis.
- Do not run chaos without blast radius.
- Do not run chaos without abort conditions.
- Do not run chaos when SLO budget is exhausted.
- Do not target primary AOIS from this curriculum.
- Treat blocked experiments as successful guardrail behavior.
- Include AI quality and safety failure modes in reliability testing.

## What The Next Version Will Build On

v19.5 builds on chaos engineering by focusing on AI failure engineering and
governance enforcement. The next question is not just "can AOIS survive a
failure?" It is "can AOIS prevent autonomous agents from crossing policy,
safety, tool, cost, or operational boundaries?"
