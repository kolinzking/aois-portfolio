# v17.5 Next Version Bridge

Authoring status: authored

## What This Version Unlocks

v17.5 turns AOIS telemetry into reliability decisions. The system can now say
whether services and agents are meeting targets, whether error budgets are being
burned too quickly, and whether risky automation should continue.

## Why The Next Version Exists

v18 exists because SLOs eventually fail.

When budgets burn, AOIS needs incident response maturity: severity levels,
runbooks, escalation rules, diagnosis flow, communication discipline, and
post-incident review.

Carry forward the v17.5 rule: an alert is useful only if it connects to a clear
operator action.
