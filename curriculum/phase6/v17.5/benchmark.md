# v17.5 Benchmark

Authoring status: authored

## Measurements

Record these values for the checkpoint:

- Validator status: `pass`
- Simulator status: `pass`
- Runtime services started: `0`
- Service SLO count: `2`
- Agent SLO count: `1`
- API achieved percentage
- Stream consumer achieved percentage
- Agent achieved percentage
- Exhausted budget count
- Repo size after checkpoint
- Virtual environment size after checkpoint
- Host memory available after checkpoint

## Interpretation

The benchmark proves that v17.5 teaches SLO math and reliability policy without
starting a monitoring stack. The important outcome is not throughput. The
important outcome is that AOIS can explain:

- what good means
- how much failure is allowed
- how fast the budget is burning
- when to alert
- when to freeze risky changes
- when to route agent actions to human review
