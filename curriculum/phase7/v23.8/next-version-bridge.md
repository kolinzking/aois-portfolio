# v23.8 Next Version Bridge

Authoring status: authored

## What This Version Unlocks

v23.8 gives AOIS-P an autonomy control surface. The system can now decide
whether to remain disabled, enter shadow mode, enter supervised mode, request
approval, demote, roll back, pause, or allow limited autonomy in policy.

This means future agent runtime work has an operating envelope instead of a
binary on/off switch.

## Why The Next Version Exists

v24 introduces multi-agent collaboration.

Once a single agent has operating modes, the next risk is coordination between
agents. Multi-agent systems need shared state, ownership, conflict resolution,
handoff boundaries, and evaluation of collaborative behavior.

v23.8 answers "how much autonomy is allowed for this agent?" v24 answers "how
do multiple agents coordinate without violating those controls?"
