# v23 Looking Forward

Authoring status: authored

## What You Should Carry Forward

A stateful loop should be boring, bounded, and explainable.

Carry forward these rules:

- stop rules come before action rules
- approval waits are respected before tool planning
- registry blocks are terminal until review
- unchanged state hashes stop the loop
- iteration limits are hard stops
- budget reserve is a control, not a report
- orchestration actions stay plan-only until runtime governance exists

## What The Next Version Will Build On

v23.5 will evaluate the connected agent behavior.

The next step is measuring whether route decisions, registry decisions,
workflow states, and orchestration actions line up with expected outcomes across
representative incidents and failure cases.
