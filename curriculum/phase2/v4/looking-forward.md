# v4 Looking Forward

Authoring status: authored

## What You Should Carry Forward

- Docker build is not the same as Dockerfile authoring
- build context affects disk and security
- localhost binding matters
- non-root runtime matters
- resource limits matter
- `aois-p` names distinguish portfolio resources

## What The Next Version Will Build On

`v5` will build on the container plan by adding security controls and threat thinking.

The goal is not only to package AOIS, but to package it in a way that is harder to misuse.
